To Do App - Oppimisprosessi ja tekeminen

Miksi To Do App?

Hyvä tapa opetella perustoiminnallisuuksia aloitteleville koodareille " Usually, the next example to come after the Hello World is the Todo List exercise " Mahdollistaa oppijalle tavan ymmärtää perusteita Pitää yllä kiinnostusta, sillä luodaan toimiva ohjelma/appi Mahdollistaa ymmärryksen siitä, ettei oppispolun tarvitse olla niin monimutkainen Yksi työkalu uusien teknologioiden oppimiseen

Mitä tehtiin?

Yksinkertaisuudessaan huikea todo-appi, joka käyttää REST-APIa ja tallettaa luodut taskit tietokantaan

Toiminnallisuus: 1) Lisää uusia taskeja 2) Tallentaa taskit tietokantaan 3) Poistaa taskeja listasta - poistaa myös taskit tietokannasta 3) Merkata taskit tehdyksi - arvo vaihtuu tietokannassa 4) Siirtää tehdyt taskit arkistoon, eli ei tarvitse poistaa

Teknologia: Back-end: NodeJs, Express, MongoDb ja Mongoose Client: Vanilla JavaScript, HTML, CSS

Mitä opittiin?

Ajankäyttö: 1) Päivä Back-endia ja crudi kuntoon, taistelua gitin kanssa 2) Päivä siihen, että käyttis ja tietokantatieto keskustelivat keskenään 3) Vika päivä backin ja frontin viilausta

Hyviä juttuja: - hyvä ryhmädynamiikka, samanlainen taitotaso - tehtiin yhdessä koko ajan, opittiin ja meillä oli hauskaa - mentiin askel kerrallaan, ongelma kerrallaan, siksi jumituttiin usein - ja ollaan parepia kuin kolmepäivää sitten? Mitä opimme - datan käsittely kannattaa tehdä back-endissä, ei frontissa - JavaScript on outoa ja ihanaa - kokeiltiin git -f komentoa, kun haluttiin palaa edelliseen committiin, joka ei tuhonnutkaan kaikkea! - debuggaus ja selaimen konsolin käyttöä - kuinka käyttöliittymä ja api keskustelee keskenään, eli kuinka Vanilla JavaScript keskustelee palvelimen kanssa Mitä olsii voinut tehdä paremmin - tehokkaampaa jos kaikki olisivat tehneet erikseen - ei tiedetty mistä ongelmaa olisi lähdetty etsimään usein - käyttis suunnittelua ja tarkoituksenmukaisuutta oisi voinut miettiä vielä pidempään nyt haluttiin vaan tehdä jotain toimivaa!
